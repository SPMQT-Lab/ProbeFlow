## Themes — issues that recur across angles

### Theme A: Calibration & metadata silently dropped through the pipeline
Multiple angles flag the same root cause — physical-unit metadata is not threaded consistently through processing operations.

- Pixel sizes not forwarded to `subtract_background(step_tolerance)` and `facet_level` ([image-proc #1](2026-05-27-image-processing.md)).
- `scan_range_m` not updated by shape-changing geometric ops ([image-proc #4](2026-05-27-image-processing.md)).
- Forward/backward channel metadata not flipped by `flip_horizontal` → doubly-mirrored backward channel on subsequent blend ([image-proc #11](2026-05-27-image-processing.md)).
- DAT→SXM converter ignores `SCAN_DIR='up'` so round-trip image is upside-down ([io #5](2026-05-27-io.md)) ✅ FIXED — `reconstruct_from_hdr_imgs` now applies `np.flipud` to each plane when `SCAN_DIR='up'`, matching the convention in `write_sxm_with_planes`. Both the standalone DAT→SXM converter and the `write_sxm` DAT-source path benefit. Regression test confirms `ScanYDirec=0` DAT roundtrip preserves display orientation.
- Createc DAT reader's `is_partial_scan` flag and decode warnings discarded by `read_dat` ([io #18](2026-05-27-io.md)).
- RHK SM4 `z_units` propagates raw — no `pm`/`Å`/`nm` → SI normalisation ([io #19](2026-05-27-io.md)).

**Recommended fix shape**: make `apply_processing_state` aware of the scan's calibration (or take it as an input dataclass) and update it for every op that changes shape or semantics.

### Theme B: FFT/window normalisation across the codebase
Four physics findings plus two image-processing findings cluster around the same window/DC-subtraction pattern. Each is locally small; together they make peak-finding thresholds and FFT magnitudes silently vendor-of-the-window.

- Biased mean subtraction on masked ROI ([physics #1](2026-05-27-physics.md), [image-proc #5](2026-05-27-image-processing.md))
- No window coherent-gain compensation feeding into Bragg peak finding ([physics #5](2026-05-27-physics.md))
- `np.hanning` zero-endpoint vs the periodic Hann ([physics #6](2026-05-27-physics.md))
- `fourier_filter` axis-normalised vs cycles/pixel ([physics #7](2026-05-27-physics.md))
- `fft_magnitude` and `fourier_filter` accept different window vocabularies ([image-proc #9](2026-05-27-image-processing.md))
- `find_bragg_peaks_in_annulus` `==` equality on `local_max_img` ([numerical #5](2026-05-27-numerical.md))

**Recommended fix shape**: one consolidated FFT helpers module under `analysis/fft_lattice.py` (also helps arch-backend finding 16 about `processing/filters.py` being 1106 LOC with mixed concerns). Use `scipy.signal.windows` with consistent vocabulary; normalise magnitudes by coherent gain; drive Bragg-peak detection from a *score* (z-score within annulus) rather than absolute log-magnitude.

### Theme C: `quantize_bit_depth` (the new 8/16-bit Image Type feature)
Four agents independently flagged this same operation from four different angles. The feature shipped recently (commit `cafb10d`) and exposes a UX promise ("256 distinct levels") that doesn't survive normal pipeline behaviour.

- Data-dependent `vmin`/`vmax` from raw extrema ([physics #3](2026-05-27-physics.md), [numerical #3](2026-05-27-numerical.md))
- Pipeline order puts quantize before `arithmetic_ops` ([image-proc #2](2026-05-27-image-processing.md))
- Quantize runs late so its input shape/extent depends on prior geometric ops ([image-proc #7](2026-05-27-image-processing.md))
- `round`-to-nearest-even bias on idempotent re-quantization ([physics #16](2026-05-27-physics.md))

**Recommended fix shape**: accept explicit `vmin`, `vmax`, `quantum` arguments, record them in the step params for reproducibility, and run quantize *after* arithmetic in the pipeline (or refuse arithmetic after quantize with a clear warning).

### Theme D: Re-emerged duplication that Stage 2 was supposed to address
Six findings from the backend-architecture angle show that Stage 2's surgical-seam approach is correct but a second pass is needed: two `MeasurementResult` dataclasses, two `FeaturePoint`, two local-maxima kernels, two `roi_statistics` variants, **six** metre→nm/Å formatters, two `ProcessingStep` dataclasses, two geometric-op naming vocabularies, four ROI→mask resolution patterns.

**Recommended fix shape**: same Stage-2 approach — pick canonical kernel in `measurements/`, remove duplicates from `analysis/`, introduce `probeflow/measurements/formatting.py` and `probeflow/measurements/roi_resolve.py` helpers.

### Theme E: GUI lifecycle / `RuntimeError` on close
The recent `f6aac4a` fix addressed one site of the C++-object-already-deleted RuntimeError pattern. The GUI agent found sibling cases:

- Seven modeless dialogs not closed by `ImageViewerDialog.closeEvent` ([gui-arch #22](2026-05-27-architecture-gui.md))
- Lattice-grid dock uses `visibilityChanged` instead of `destroyed` for cleanup ([gui-arch #3](2026-05-27-architecture-gui.md))
- `ROIManagerDock` doesn't set its own `objectName` ([gui-arch #5](2026-05-27-architecture-gui.md))
- `_open_viewers` weak-keepalive pattern brittle ([gui-arch #13](2026-05-27-architecture-gui.md))

**Recommended fix shape**: collect modeless dialog refs into a `self._modeless_children: list[QDialog]` list, standardise on `WA_DeleteOnClose=True` for non-result dialogs, use `QObject.destroyed` (not `visibilityChanged`) for dock lifecycle hooks.

### Theme F: Silent-failure UX patterns
Across all six reports, the same shape recurs: a function silently degrades (drops a warning, returns a default, falls through a fallback branch) instead of surfacing the issue.

- `_detect_channel_count` overrides header `Channels` silently ([io #3](2026-05-27-io.md))
- Nanonis spec `_parse_header_float` coerces missing tip position to `(0.0, 0.0)` ([io #4](2026-05-27-io.md)) ✅ FIXED — `_parse_header_float` now returns NaN for missing/malformed values. NaN propagates through `spec_position_to_pixel` (returns None) so GUI overlays correctly skip drawing a marker. CLI/spec-viewer formatters print "nan" instead of silently showing the origin.
- `read_all_sxm_planes` truncates plane list to a warning but `Scan` does not record it ([io #10](2026-05-27-io.md))
- `gui_adapter.processing_state_from_gui` silently squashes failed step coercions ([image-proc #16](2026-05-27-image-processing.md))
- `apply_outlier_mask` returns empty arrays when all input is non-finite ([numerical #14](2026-05-27-numerical.md))
- `gmm_autoclip` returns clamped percentiles silently on degenerate data ([numerical #9](2026-05-27-numerical.md))

**Recommended fix shape**: prefer warnings (`warnings.warn(UserWarning)`) attached to `Scan.warnings` or `MeasurementResult.warnings` over silent fallback. Surface them in provenance sidecars and the GUI status bar.

## Architectural posture

The backend layering is genuinely clean — outside `cli/commands/gui.py`, no backend module imports `probeflow.gui` (verified by the architecture agents independently). The Stage 2 seams (`measurements/adapters.py`, `gui/roi_context.py`, `analysis/lattice_correction_workflow.py`) are correctly used by code added since. **The trouble is in the gaps:**

1. The aspirational `ScanGraph` API is fully implemented but unused, and every subpackage boundary docstring promotes it as canonical.
2. The `plugins/` scaffolding (127 LOC, 5 files including an empty `adapters/` subpackage) signals a plugin system that doesn't actually load anything.
3. The new GUI dialogs correctly delegate scientific logic to backend modules, but the floating Feature Counting window (recent `82cd4d8`/`58636e7`) duplicates ~150 LOC of orchestration from `ProbeFlowWindow` and shares signal objects across worker spawns ([gui-arch #1 + #2](2026-05-27-architecture-gui.md)).

The package is at the size where ambiguity about "which seam is canonical" costs real contributor time. None of the architectural issues block users; all are amenable to surgical-seam fixes.

## Suggested order of attack

Sized to roughly Stage-2-style bounded slices. Earlier items unblock or reduce the cost of later items.

1. **Fix the S0 polar-decomposition** ([numerical #1](2026-05-27-numerical.md)). Small change, removes a silent reflective transform from the lattice-correction surface. Includes [numerical #4](2026-05-27-numerical.md) and [physics #12](2026-05-27-physics.md) as dependents.
2. **Silent-data-corruption IO trio**: SXM writer overwrite-source guard ([io #1](2026-05-27-io.md)), ROI sidecar path collapse ([io #2](2026-05-27-io.md)), Createc channel-count override ([io #3](2026-05-27-io.md)). One commit each, all S1, all directly user-affecting.
3. **Thread calibration through `apply_processing_state`** ([image-proc #1 + #4](2026-05-27-image-processing.md)). Single design change ("the function takes a scan-calibration context"); fixes the silent-step-tolerance degradation and the wrong-pixel-size-after-shape-change cluster. Worth doing before any new processing op is added.
4. **Quantize_bit_depth rework** ([physics #3](2026-05-27-physics.md), [numerical #3](2026-05-27-numerical.md), [image-proc #2 + #7](2026-05-27-image-processing.md)). Accept explicit `vmin`/`vmax`/`quantum`, record in step params, move execution to after arithmetic. Closes Theme C.
5. **FFT/window consolidation** ([physics #1 + #5 + #6 + #7](2026-05-27-physics.md), [image-proc #5 + #9](2026-05-27-image-processing.md), [arch-backend #16](2026-05-27-architecture-backend.md)). Splits `processing/filters.py` into a real filters file + `analysis/fft_lattice.py`, normalises window choice, fixes DC subtraction. Single moderately-sized refactor; closes Theme B and starts to address the 1106-LOC filters monolith.
6. **GUI dialog-lifecycle audit** ([gui-arch #3 + #22](2026-05-27-architecture-gui.md), plus minor [#5](2026-05-27-architecture-gui.md)). Adopt `_modeless_children` list pattern; standardise `WA_DeleteOnClose=True` for non-result dialogs; switch from `visibilityChanged` to `destroyed`. Closes Theme E.
7. **Stage-2 duplication second pass** ([arch-backend #1–#5 + #10–#12](2026-05-27-architecture-backend.md)). Remove dead `analysis.measurements.MeasurementResult`, merge `FeaturePoint`, pick canonical local-maxima kernel, build `measurements/formatting.py` and `measurements/roi_resolve.py`. Largest slice; non-blocking; pays off cumulatively.
8. **Decide and document `ScanGraph` and `plugins/`** ([arch-backend #6 + #7](2026-05-27-architecture-backend.md)). Wire-in or hide-under-underscore. Either way, update the boundary-rules docstrings so they describe the actual architecture.
9. **FeatureCounting controller extraction** ([gui-arch #1 + #2](2026-05-27-architecture-gui.md)). Removes the ~150 LOC duplication and fixes the worker-signals race in one go.

## Process notes

- Two parallel agent batches stalled silently at the harness level. The eventual approach that worked was sequential agents for the last three subpackages (IO, backend architecture, GUI architecture). This is worth knowing for future review passes — for the same-shape work, plan on sequential execution.
- Skeleton-first reports (instructing each agent to write its report file as the first tool call) preserved partial progress when agents stalled. Recommended for any future long-running agent task.
- One agent's completion notification was lost (physics agent — its report exists on disk but no `<task-notification>` arrived). Tracking by JSONL file mtime + report-on-disk presence is the reliable status signal, not the notification channel.
